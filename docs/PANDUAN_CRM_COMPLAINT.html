Panduan Penggunaan Sistem CRM Complaint

Versi dokumen: 1.0 | Tanggal: 2 Maret 2026

Panduan ini menjelaskan cara menggunakan sistem CRM Complaint untuk proses penerimaan, investigasi, CAPA, approval, hingga penutupan ticket customer complaint.

1. Tujuan Sistem

  • Mencatat keluhan customer secara terstruktur dan terdokumentasi.
  • Memastikan tindak lanjut complaint sesuai SLA.
  • Menerapkan workflow CAPA: QA -> Manager -> Closed.
  • Menyediakan jejak audit (audit log) untuk setiap aksi penting.

2. Role dan Akses

Role Hak Akses Utama
Admin Akses penuh semua fitur.
QA Create ticket, update status, submit CAPA, close ticket (jika CAPA approved).
Manager Review CAPA, approve/reject CAPA, close ticket.
CS Create ticket, tambah catatan, upload lampiran.
Viewer View dashboard/list dan export laporan.

3. Alur Kerja Harian

  1. Login ke sistem menggunakan akun yang diberikan administrator.
  2. Buka menu Complaint untuk melihat daftar ticket.
  3. Klik Input Ticket Baru untuk membuat complaint baru.
  4. Isi data lengkap customer dan detail keluhan, lalu simpan.
  5. Buka halaman detail ticket untuk investigasi, update status, dan upload bukti.
  6. QA mengisi dan submit CAPA.
  7. Manager approve/reject CAPA.
  8. Jika CAPA approved, lakukan close ticket dengan ringkasan final.

4. Penggunaan Halaman Complaint

4.1 Halaman List Complaint

  • Fitur filter: keyword, status, severity, brand, rentang tanggal.
  • Fitur export: Export Excel dan Export PDF.
  • Klik Detail untuk membuka halaman detail ticket.

4.2 Halaman Input Ticket Baru

Field minimum yang wajib: Nama Customer, Channel, Tanggal Complaint, Severity, Status, dan Deskripsi Keluhan.
  • Isi Brand, Kategori, Kode Produksi, PIC, dan Target Resolution bila tersedia.
  • Upload lampiran bukti (jpg/png/webp/mp4/pdf), maksimal 5 file, 10MB per file.
  • Simpan ticket, sistem otomatis membuat nomor ticket unik.

4.3 Halaman Detail Ticket

  • Menampilkan data utama ticket, status, CAPA status, timeline update, lampiran, dan audit log.
  • Update Status: gunakan untuk perubahan status operasional (Open/Investigating/Action Plan/Resolved).
  • Catatan: menambah progress investigasi.
  • Lampiran: upload/download bukti pendukung.

5. Workflow CAPA (QA -> Manager -> Closed)

Langkah QA:

  1. Isi Root Cause.
  2. Isi Corrective Action.
  3. Isi Preventive Action.
  4. Tentukan CAPA Due Date.
  5. Submit CAPA ke Manager.

Langkah Manager:

  1. Review CAPA di halaman detail ticket.
  2. Approve jika valid, atau Reject dengan alasan.
  3. Jika reject, QA wajib revisi dan submit ulang.

Penutupan Ticket: ticket hanya bisa di-close jika CAPA sudah Approved. Saat close, isi ringkasan final penyelesaian.

6. Audit Log

Setiap aksi penting user direkam otomatis, termasuk:

  • Create ticket
  • Update status
  • Submit/approve/reject/close CAPA
  • Tambah catatan
  • Upload/download lampiran
  • Export Excel/PDF

Audit log digunakan untuk kebutuhan traceability, kontrol internal, dan pemeriksaan kepatuhan.

7. Dashboard dan Monitoring KPI

  • Total Complaint
  • Open Ticket
  • Closed 30 Hari
  • Average Resolution Time
  • SLA Compliance
  • Distribusi status dan top brand complaint

8. Panduan Export Laporan

  • Gunakan filter terlebih dahulu pada halaman list complaint.
  • Klik Export Excel untuk file CSV kompatibel Excel.
  • Klik Export PDF untuk tampilan print-ready.
  • Simpan hasil export sesuai periode pelaporan yang dibutuhkan.

9. Troubleshooting Singkat

Masalah Tindakan
Tidak bisa akses menu tertentu Cek role akun. Hubungi Admin untuk perubahan hak akses.
Ticket tidak bisa di-close Pastikan CAPA sudah Approved dan ringkasan final diisi.
Upload lampiran gagal Periksa format file dan ukuran maksimal 10MB per file.
Export kosong Cek filter tanggal/status. Ulangi export tanpa filter untuk verifikasi.

10. Kontak Internal

Untuk bantuan operasional atau kendala teknis, hubungi:

  • Admin Sistem CRM
